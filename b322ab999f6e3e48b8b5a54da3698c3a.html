Ordem de Compra

UMLUseCase
Untitled :: UseCases :: Ordem de Compra

Description

FLUXO PRINCIPAL

  1. Usuário seleciona ativo.
  2. Usuário escolhe tipo de ordem a mercado. [FA1]
  3. Usuário seleciona quantidade de ativos. [FA5]
  4. Usuário envia ordem à corretora.
  5. Corretora valida e aceita ordem de compra. [FA2]
  6. Corretora envia ordem à B3. [FA4]
  7. B3 valida ordem de compra. [FA3]
  8. B3 executa ordem de compra e envia confirmação para a corretora.
  9. Usuário recebe confirmação na tela.
  10. Usuário recebe confirmação por email e SMS.

FA1

2.1 Usuário escolhe gatilho.
2.2 Usuário escolhe condição para o envio da ordem de compra.
2.3 Caso condição escolhida ocorra, ordem de compra é enviada para a B3.

FA2

5.1 Corretora recusa ordem de compra.
5.2 Usuário recebe notificação informando motivo da recusa.

FA3

7.1 B3 recusa ordem de compra e notifica corretora.
7.2 Usuário recebe notificação informando motivo da recusa.

FA4

6.1 Usuário pode modificar ordem de compra.
6.2 Corretora atualiza ordem e retorna à validação.

FA5

3.1 Usuário pode simular uma ordem de compra.
